화면에 표시되는 모든 텍스트에 다국어 지원을 적용한다.

## 트리거

- 신규 기능 개발 후 l10n 누락 확인
- 릴리즈 전 l10n 전체 점검
- 사용자가 l10n 작업을 요청할 때

## 점검 대상

| 대상          | 예시                   |
| ------------- | ---------------------- |
| UI 텍스트     | 버튼, 라벨, 제목, 설명 |
| 에러 메시지   | validation, API 에러   |
| placeholder   | input, textarea        |
| alt 텍스트    | 이미지 설명            |
| title/tooltip | 마우스오버 텍스트      |
| 확인/취소     | 다이얼로그 버튼        |

## 제외 대상

| 대상        | 이유                  |
| ----------- | --------------------- |
| 로그 메시지 | 개발자용, 번역 불필요 |
| 코드 주석   | 개발자용              |
| 테스트 코드 | 내부용                |
| 고유명사    | 브랜드명, 제품명 등   |

## 파일 구조

```text
/src/locales/
├── ko.json          # 한국어 (기본 언어)
├── en.json          # 영어
├── ja.json          # 일본어
└── index.ts         # 로케일 로더
플랫폼별 표준이 있는경우 해당 표준을 따른다 : 예 (swift의 .strings, android의 strings.xml)
```

## 실행 순서

### Phase 1: 현황 파악

1. `/src/locales/` 디렉토리 구조 확인
2. 지원 언어 목록 확인 (ko.json, en.json, ja.json 등)
3. 기본 언어 파일(ko.json)의 키 목록을 기준으로 삼음
4. l10n 설정이 없으면 위 파일 구조로 새로 구성

### Phase 2: 누락 항목 식별

1. 하드코딩된 텍스트 검색
2. 신규 추가된 컴포넌트 확인
3. **언어 파일 간 키 불일치 검사** (기본 언어 기준)
4. 누락 항목 목록 작성

### Phase 3: 번역 키 추가

1. 번역 키 생성 (네이밍 규칙 준수)
2. **모든 언어 파일에 동일 키 추가** (누락 방지)
3. 컴포넌트에서 번역 키 사용으로 교체

### Phase 4: 검증

1. **키 일관성 검사**: 모든 언어 파일의 키가 동일한지 확인
2. 각 언어별 화면 확인
3. 텍스트 잘림/오버플로우 확인
4. 누락 항목 재확인

## 번역 키 네이밍 규칙

```text
<domain>.<page>.<element>.<description>

예시:
- common.button.save
- common.button.cancel
- auth.login.title
- auth.login.error.invalidPassword
- dashboard.chart.noData
```

## 언어 파일 형식

```json
// ko.json (기본 언어)
{
  "common.button.save": "저장",
  "common.button.cancel": "취소",
  "auth.login.title": "로그인"
}

// en.json
{
  "common.button.save": "Save",
  "common.button.cancel": "Cancel",
  "auth.login.title": "Login"
}

// ja.json
{
  "common.button.save": "保存",
  "common.button.cancel": "キャンセル",
  "auth.login.title": "ログイン"
}
```

```markdown

### 추가된 번역 키

| 키               | ko     | en    | ja       |

| auth.login.title | 로그인 | Login | ログイン |

### 수정된 파일

- /src/locales/ko.json

```

## 주의사항

- **키 추가/삭제 시 모든 언어 파일 동시 수정** (필수)
- 기본 언어(ko.json)를 기준으로 다른 언어 파일 동기화
- 번역 키 삭제 전 사용처 확인
- 번역이 없는 경우 기본 언어 값 또는 키 자체를 fallback으로 표시
- 커밋 메시지: `feat: add l10n for <feature>`
