화면에 표시되는 모든 텍스트에 다국어 지원을 적용한다.

## 트리거

- 신규 기능 개발 후 l10n 누락 확인
- 릴리즈 전 l10n 전체 점검
- 사용자가 l10n 작업을 요청할 때

## 점검 대상

| 대상 | 예시 |
|------|------|
| UI 텍스트 | 버튼, 라벨, 제목, 설명 |
| 에러 메시지 | validation, API 에러 |
| placeholder | input, textarea |
| alt 텍스트 | 이미지 설명 |
| title/tooltip | 마우스오버 텍스트 |
| 확인/취소 | 다이얼로그 버튼 |

## 제외 대상

| 대상 | 이유 |
|------|------|
| 로그 메시지 | 개발자용, 번역 불필요 |
| 코드 주석 | 개발자용 |
| 테스트 코드 | 내부용 |
| 고유명사 | 브랜드명, 제품명 등 |

## 실행 순서

### Phase 1: 현황 파악

1. 기존 l10n 파일 구조 확인
2. 지원 언어 목록 확인
3. 번역 키 네이밍 규칙 확인

### Phase 2: 누락 항목 식별

1. 하드코딩된 텍스트 검색
2. 신규 추가된 컴포넌트 확인
3. 누락 항목 목록 작성

### Phase 3: 번역 키 추가

1. 번역 키 생성 (네이밍 규칙 준수)
2. 기본 언어(ko/en) 번역 추가
3. 컴포넌트에서 번역 키 사용으로 교체

### Phase 4: 검증

1. 각 언어별 화면 확인
2. 텍스트 잘림/오버플로우 확인
3. 누락 항목 재확인

## 번역 키 네이밍 규칙

```
<domain>.<page>.<element>.<description>

예시:
- common.button.save
- common.button.cancel
- auth.login.title
- auth.login.error.invalidPassword
- dashboard.chart.noData
```

## 결과물

작업 완료 후 ORDER 파일에 기록:

```markdown
## l10n 작업 내역

### 추가된 번역 키
| 키 | ko | en |
|----|----|----|
| auth.login.title | 로그인 | Login |

### 수정된 파일
- /src/pages/Login.tsx
- /src/locales/ko.json
- /src/locales/en.json
```

## 주의사항

- 기존 번역 키 수정 시 모든 언어 파일 동시 수정
- 번역 키 삭제 전 사용처 확인
- 커밋 메시지: `feat: add l10n for <feature>`
