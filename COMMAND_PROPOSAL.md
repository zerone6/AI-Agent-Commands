# 추가 커맨드 제안

일반적인 개발 방법론을 기반으로 한 추가 커맨드 제안 목록

---

## 현재 커맨드 커버리지

```
계획 → 실행 → 중단 → 종료 → Sprint 종료
       ↓
   리팩토링/l10n
```

---

## 제안 커맨드 목록

### 1. 코드 리뷰 (Code Review)

| 항목 | 설명 |
|------|------|
| 용도 | PR 전 코드 품질 점검 |
| 트리거 | ORDER 완료 후, 머지 전 |
| 커밋 타입 | - (리뷰만) |

점검 항목:
- 코딩 컨벤션 준수
- 타입 안전성
- 에러 처리
- 성능 이슈
- 보안 취약점

---

### 2. 버그 수정 (Hotfix)

| 항목 | 설명 |
|------|------|
| 용도 | 긴급 버그 수정 (ORDER 없이 빠른 대응) |
| 트리거 | P0/P1 버그 발생 |
| 커밋 타입 | fix |

특징:
- ORDER 작성 생략 가능
- 간소화된 문서
- 빠른 머지

---

### 3. 의존성 업데이트 (Dependency Update)

| 항목 | 설명 |
|------|------|
| 용도 | 패키지 버전 업데이트 및 보안 패치 |
| 트리거 | 정기 점검, 보안 취약점 발견 |
| 커밋 타입 | chore |

점검 항목:
- outdated 패키지
- 보안 취약점 (npm audit)
- breaking changes 확인

---

### 4. 테스트 작성 (Test Coverage)

| 항목 | 설명 |
|------|------|
| 용도 | 테스트 커버리지 향상 |
| 트리거 | 커버리지 목표 미달, 릴리즈 전 |
| 커밋 타입 | test |

대상:
- 단위 테스트
- 통합 테스트
- E2E 테스트

---

### 5. 문서화 (Documentation)

| 항목 | 설명 |
|------|------|
| 용도 | API 문서, README, 가이드 작성/업데이트 |
| 트리거 | 신규 기능 추가 후, 릴리즈 전 |
| 커밋 타입 | docs |

대상:
- API 명세
- 컴포넌트 스토리북
- 설치/배포 가이드

---

### 6. 성능 최적화 (Performance)

| 항목 | 설명 |
|------|------|
| 용도 | 로딩 속도, 번들 크기, 메모리 최적화 |
| 트리거 | 성능 이슈 발견, 릴리즈 전 |
| 커밋 타입 | perf |

점검 항목:
- 번들 사이즈 분석
- 불필요한 리렌더링
- 이미지/에셋 최적화
- 레이지 로딩

---

### 7. 환경 설정 (Environment Setup)

| 항목 | 설명 |
|------|------|
| 용도 | 개발/스테이징/프로덕션 환경 설정 |
| 트리거 | 신규 환경 구축, 설정 변경 |
| 커밋 타입 | chore |

대상:
- 환경변수 설정
- CI/CD 파이프라인
- Docker 설정

---

### 8. 릴리즈 (Release)

| 항목 | 설명 |
|------|------|
| 용도 | 버전 태깅 및 배포 준비 |
| 트리거 | Sprint 종료 후 배포 시점 |
| 커밋 타입 | chore |

절차:
- CHANGELOG 작성
- 버전 번호 업데이트
- 태그 생성
- 배포 체크리스트

---

## 우선순위 제안

| 순위 | 커맨드 | 이유 |
|------|--------|------|
| 1 | Hotfix | 긴급 대응 필수 |
| 2 | Code Review | 품질 관리 |
| 3 | Release | 배포 프로세스 표준화 |
| 4 | Test Coverage | 안정성 확보 |
| 5 | Dependency Update | 보안/유지보수 |
| 6 | Performance | 사용자 경험 |
| 7 | Documentation | 협업 효율 |
| 8 | Environment Setup | 필요시 추가 |

---

## 전체 개발 사이클 (제안)

```
[계획]
  ORDER 작성 -> 리뷰 -> 승인

[개발]
  ORDER 실행 -> (중단 시 SAVETEMP) -> ORDER 종료

[품질]
  Code Review -> Test -> Refactoring -> l10n

[배포]
  Sprint 종료 -> Release

[유지보수]
  Hotfix -> Dependency Update -> Performance
```
