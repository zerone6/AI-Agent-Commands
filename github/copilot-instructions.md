# 기본 룰
- 모든 사고 과정과 설명은 한국어로 한다.
- 새로운 텍스트가 Text가 추가되면 i18n 처리를 필수로한다. 소스에 하드코딩으로 명칭을 넣지 않는다.
- i18n 번역 파일 위치 : `frontend/src/i18n/translations.js'
- GIT 커밋 메시지는 -m옵션을 사용하지 않는다. /gitlog.txt 파일을 생성하고 그 파일에 상세 로그를 작성한후 사용한다.
- 새로운 작업은 항상 doc폴더에 작업 계획서를 만들어서 기록한다. 해당 문서의 작업이 완료되면 문서의 작업 결과와 그에 따른 TODO도 문서에 반영한다
- ORDERXXXX.md는 새로운 작업에 대한 문서
- REPACTORINGXXXX.md는 리팩토링에 대한 문서
